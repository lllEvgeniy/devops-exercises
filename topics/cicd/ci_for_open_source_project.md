## CI для проекта с открытым исходным кодом

1. Выберите проект с открытым исходным кодом на Github и создайте его форк.
2. Создайте конвейер/рабочий процесс CI для проекта, который вы разветвили.
3. Конвейер/рабочий процесс CI будет включать в себя все, что имеет отношение к разветвленному вами проекту. Например:
  * Если это проект Python, вы запустите PEP8.
  * Если в проекте есть каталог модульных тестов, вы будете запускать эти модульные тесты как часть CI.
4. В отдельном файле опишите, что работает как часть CI и почему вы решили его включить. Вы также можете описать любые мысли, дилеммы, проблемы, которые у вас возникли.

### Бонус

Контейнеризируйте приложение разветвленного проекта, используя любой контейнерный движок (например, Docker, Podman).<br>
После успешного запуска приложения в контейнере отправьте Dockerfile в исходный проект (но будьте готовы к тому, что сопровождающему это может не понадобиться/не захотеть).
