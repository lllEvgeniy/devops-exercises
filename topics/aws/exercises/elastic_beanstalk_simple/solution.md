## AWS Elastic Beanstalk — Node.js

### Предварительные условия
1. убедитесь, что приложение node.js имеет команду _npm start_, указанную в файле __package.json__, как показано в следующем примере.

```
{

  "name": "application-name",
  "version": "0.0.1",
  "private": true,
  "scripts": {
    "start": "node app"
  },
  "dependencies": {
    "express": "3.1.0",
    "jade": "*",
    "mysql": "*",
    "async": "*",
    "node-uuid": "*"
} 

```

2. заархивируйте приложение и убедитесь, что не заархивированы родительская папка, а только файлы вместе, например:

```
\Parent - (exclude the folder itself from the the zip)
- file1 - (include in zip)
- subfolder1 (include in zip)
- file2 (include in zip)
- file3 (include in zip)
```

### Решение

1. Создайте «новую среду»
2. Выберите Среда => _Среда веб-сервера_.
3. Заполните раздел «Создание среды веб-сервера».
а. Заполните «Имя приложения»
4. Заполните раздел «Информация об окружающей среде».
а. Заполните «Имя среды»
б. Домен — «Оставить для автоматически созданного значения»
5. Платформа
а. Выберите Платформу => _node.js_.
6. Код приложения => загрузите ZIP-код со своего локального компьютера.
7. Создайте среду
8. Подождите, пока появится окружение.
9. Проверьте сайт
а. Перейдите на вкладку _Приложения_, 
б. выберите недавно созданное приложение node.js
в. нажмите на URL - выделено
